Goal: After training, your model should achieve at least 80% accuracy on a validation set within 20 epochs. (Or as close as possible as long as there is demonstrated effort to achieve this goal.)

Data split The training set should consist of 40000 images, the validation set should consist of 10000 images, and the test set should consist of the remaining 10000 images. Please use the Keras load_data() function to import the data set.

Implementation All code should be in Keras/python.

The CIFAR-10 dataset¶

  • The CIFAR-10 dataset consists of 60000 32x32 colour images in 10 classes, with 6000 images per class.
  • There are 50000 training images and 10000 test images.
  • The dataset is divided into five training batches and one test batch, each with 10000 images.
  • The test batch contains exactly 1000 randomly-selected images from each class.
  • The training batches contain the remaining images in random order, but some training batches may contain more images from one class than another
  • Between them, the training batches contain exactly 5000 images from each class

def fn_Train(modelname, model, p_optimizer, p_loss, p_metrics, p_train_data, p_train_labels, p_epochs, p_batch_size, p_val_data, p_val_labels, p_verbose,p_test_data, p_test_labels):
    print('Modelname:',modelname)
    print('Optimizer:', p_optimizer)
    print('Loss',p_loss)
    print('Metrics',p_metrics)    

    early_stop = EarlyStopping(monitor = 'val_accuracy', patience = 3)
    
    start_time = time.time()
    
    print('Model Compile with optimizer, loss function and evaluation method')
    model.compile(optimizer=p_optimizer,
                loss=p_loss,
                metrics=[p_metrics])
    
    print('Model Training to see the tensor shapes and number of parameters for each layer')
    # The values for the training loss and accuracy, as well as the validation loss and accuracy, are stored in the history variable
    history = model.fit(p_train_data, 
                    p_train_labels, 
                    epochs = p_epochs, 
                    batch_size = p_batch_size, 
                    validation_data=(p_val_data, p_val_labels),
                    verbose = p_verbose)
    print('Total Training Time (second):', time.time() - start_time)
    
    print('Visualize the progress our network makes on the loss and accuracy as the number of epochs increases.')

    history.history.keys()
    history_dict = history.history # the dictionary that has the information on loss and accuracy per epoch

    loss_values = history_dict['loss']   # training loss
    val_loss_values = history_dict['val_loss'] # validation loss

    epochs = range(1, len(loss_values)+1)  #creates list of integers to match the number of epochs of training

    # code to plot the results
    plt.plot(epochs, loss_values, 'b', label="Training Loss")
    plt.plot(epochs, val_loss_values, 'r', label="Validation Loss")
    plt.title("Training and Validation Loss")
    plt.xlabel("Epochs")
    plt.xticks(epochs)
    plt.ylabel("Loss")
    plt.legend()
    plt.show()

    # As above, but this time we want to visualize the training and validation accuracy
    acc_values = history_dict['accuracy']
    val_acc_values = history_dict['val_accuracy']

    plt.plot(epochs, acc_values, 'b', label="Training Accuracy")
    plt.plot(epochs, val_acc_values, 'r', label="Validation Accuracy")
    plt.title("Training and Validation Accuracy")
    plt.xlabel("Epochs")
    plt.xticks(epochs)
    plt.ylabel("Accuracy")
    plt.legend()
    plt.show()
    

    print('Prediction using test set.')    
    test_loss, test_acc = model.evaluate(test_data,  test_labels, verbose=p_verbose)
    print(test_acc)

Part 2: Describe What you did (50% of grade)¶

1. At frist, we came up with the model which is derived from the lab of "convolutions" for MNIST dataset. Below is the the first model.¶

model_1 = Sequential([
Conv2D(filters=32, kernel_size=(3, 3), strides=1, padding='same', activation='relu',input_shape=input_shape),
Conv2D(filters=32, kernel_size=(3, 3), strides=2, padding='valid',activation='relu'),
Conv2D(filters=64, kernel_size=(3, 3), strides=1, padding='same', activation='relu'),
Conv2D(filters=64, kernel_size=(3, 3), strides=2, padding='valid',activation='relu'),
Flatten(),
Dense(128, activation='relu'),
Dense(10, activation='softmax')
])

  • We decided to start with the model from our lesson to understand about the structure and approach of convolution neuron network.
  • We investigate to understand about the paramenters including filter, kernel_size, stride, padding in the convolution layer. 
      - filter = 32 / 64: the number of output filter
  - kernel_size = (3,3): the size (heigh and width) of the filter that run over the image
  - stride = 1: the filter scan/move forward 1 stride each step
  - padding-"same"/"valid": add pixel around the image to avoid missing image features
  • Then, we went through some activation functions, which are used to change the weight, learning rate, etc of neuron network to reduce loss, include sigmoid, relu, tanh. We decide to work with "relu" because we understand that its below algorithm return values in the range [0,1], this help our model easy to reach linear.
  • The first things we would like to try is optimizers. We tried to combine this model with 3 different optimizers including "rmsprop", "adam", and "sgd" to see the affect. Moreover, we use the loss function "categorical_crossentropy", evaluation function "accuracy".
  • As request, the model is trained with in 20 epochs. The model is trained with the batch size = 256 for 40,000 images of train set, and validated using 10,000 images.
  • The results are like that: 
      - "rmsprop": Total training time is 762 seconds. Validation loss descrease from the first epoch down to the 5th, then it increased from 1.03 to 2.36 from the 5th to the epoch 20th. The accuracy started from 0.32 (32%), then reached the max accuracy at 0.678 (67,8%)
  - "adam" : Total training time is 761. Validation loss increase from 1.1067 to 2.23. The accuracy started from 0.42 to 0.65
  - "SGD" : Total training time is 698. Validation loss steadily went from 2.28 down to 1.35. The accuracy increased from 0.17, and got peak at 0.51.
  • We concluded that 
      - "rmsprop" and "adam" nearly worked same. Loss and accuracy in training and validating phases have the same shapes when visualizing.
  - Although the training phase is good, which strongly reduces the loss and increases rapidly the accuracy, both the loss and accuracy have the trend of increasing for validating phase. We assumed that if we try with more epochs, the model may be overfitting.
  - "SGD" is not the good optimizer for CIFAR 10 because the model learnt slowly
  • We decided to create other models

2. We created a new model that is combined using optimizers"rmsprop" and "adam". We didn't use "SGD" because we concluded it's not suitable for CIFAR-10 dataset. Below is the the second model.¶

model = Sequential([
Conv2D(filters=32, kernel_size=(3, 3), strides=1, padding='same', activation='relu', input_shape=input_shape), ##half padding
Conv2D(filters=32, kernel_size=(3, 3), strides=1, padding='same', activation='relu'),
MaxPooling2D(pool_size = (2,2)),
Dropout(0.2),
Conv2D(filters=64, kernel_size=(3, 3), strides=1, padding='valid', activation='relu'), ## no paading
Conv2D(filters=64, kernel_size=(3, 3), strides=1,padding='valid', activation='relu'),
MaxPooling2D(pool_size = (2,2)),
Dropout(0.2),
Conv2D(filters=128, kernel_size=(3, 3), strides=1, padding='valid', activation='relu'),
Conv2D(filters=128, kernel_size=(3, 3), strides=1, padding='valid', activation='relu'),
MaxPooling2D(pool_size = (2,2)),
Dropout(0.2),
Flatten(),
Dense(128, activation='relu'),
Dense(64, activation='relu'),
Dense(10, activation='softmax')
])

  • The significant change in this model is adding Pooling and Dropout layers into neuron network. The purpose of adding pooling layer is to reduce the size of input volume, and lead to reducing of hyperparameter and computing in the network. The purpose of Dropout layer is to reduce overfitting that we want to prevent from the first model.
  • Moreover, we used different filter values. And, we used half padding for the first layer only.
  • The results are as below 
      - "rmsprop": Total training time is 1778 seconds. Validation loss descrease from 1.58 down to 0.77 while the accuracy started from 0.42 (32%), then reached the max accuracy at 0.73 (73%)
    - "adam" : Total training time is 1804. Validation loss steadily descreases from 1.63 to 0.63 (red line) whereas the accuracy started from 0.39, then reached the highest accuracy 0.78 from the epoch 19th. The trend of loss and accuracy line of validation phase is strong linear. This proved that this model is learnt well. This is the best model we gained. The evaluation from testing phase is 0.778 (77,8%).

3. With our best model, the batch_size is 256. We continued to try using different batch_size for our best model. We expected to achieve the better accuracy. Below is the result¶

  • Batch_size = 64, the shape of loss and accuracy of validation phases didn't change, however, the accuracy for validation and test were 0.76 (76%) and total training time is 2065. It's not better than above final model.

4. Conclusion¶

Regarding to above experiment, we conclude that

  • "adam" and "rmsprop" optimizers work nearly the same for CIFAR-10 dataset
  • Adding pooling layer can reduce the size of input volumn (We can see detail in the "output shape" of the model summary above), this help to reduce computing in the network.
  • Adding Dropout layer can reduce the number of connection of two layers, and reduce overfitting

  • Properly choosing the batch-size is important. As our observation, batch size may impact to the training performance

  • If we have more time, we would like to try with different loss functions, activation function, kernel size (5x5, 7x7, 11x11...), batch size and learning rate. Moreover, we will try to understand clearer how to calculate the weight and size of input and output.
